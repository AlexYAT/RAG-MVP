# Title

Environment Variables Loading (dotenv) - MVP Implementation

## Goal

Реализовать автоматическую загрузку `.env` в development-режиме с минимальными изменениями в текущем MVP-контуре.

## Scope

**Включено:**

- добавление зависимости `python-dotenv`;
- добавление вызова `load_dotenv()`;
- минимальная интеграция в точку входа приложения.

**Не включено:**

- рефакторинг всей конфигурационной подсистемы;
- секрет-менеджеры;
- сложная логика профилей окружений.

## Steps

1. Добавить зависимость `python-dotenv` в backend-зависимости.
2. В точке входа приложения:
   - импортировать `load_dotenv`;
   - вызвать `load_dotenv()` до чтения переменных окружения в runtime.
3. Проверить, что:
   - переменные из `.env` доступны приложению;
   - поведение system env не нарушено (системные переменные продолжают работать).

## Definition of Done

- `OPENAI_API_KEY` доступен приложению без ручного `export` в development-сценарии.
- Приложение запускается без ошибок после добавления `dotenv`.
- `.env` корректно читается при старте.

## Risks

- конфликт значений между `.env` и системными переменными;
- неправильный порядок загрузки, при котором часть модулей читает env до `load_dotenv()`.

## Non-goals

- production-конфигурация;
- docker/env orchestration.
