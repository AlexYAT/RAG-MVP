## Title

Demo-first RAG MVP для медицинского оборудования

## Goal

* Создаётся demo-ready RAG-система.
* Цель: демонстрация, обучение и portfolio-ready результат.
* Система помогает с поиском по документации, FAQ и подбором оборудования.

## Target User

* Основной пользователь: менеджер по продажам / работе с клиентами в домене медицинского оборудования.
* Работает по сценариям.
* Допускается упрощённый UX.

## Core Scenarios

### FAQ

* Ответы на вопросы.
* Поиск по документации.

### Equipment Selection

* Уточнение требований.
* Предложение вариантов.
* Объяснение выбора.

### Equipment Overview

* Структурированная информация по направлениям.

## Development Principles

* Demo-first.
* Минимально достаточный scope.
* Декларация возможностей вместо полной реализации.
* Приоритет ясной архитектуры над преждевременной реализацией.

## High-level Capabilities

* Обработка пользовательского запроса.
* Определение сценария.
* Поиск релевантной информации.
* Формирование ответа.
* Поддержка уточняющего диалога.
* Работа через Web UI.

## Constraints

* Синтетический dataset.
* Отсутствие production требований.
* Нет масштабирования.
* Нет ролей пользователей.
* Ограниченный UX.
* Web UI обязателен для демонстрации.

## Non-goals

В MVP не входят:

* Hybrid search.
* Reranking.
* Caching.
* Policy layer.
* Сложная фильтрация / taxonomy.
* Production-ready безопасность.
* Масштабирование и SLA.

## Success Criteria

MVP считается успешным, если:

* Поддерживаются 3 сценария: FAQ, подбор, обзор.
* Есть ingest данных.
* Работает retrieval и генерация ответа.
* Есть минимальный Web UI.
* Система демонстрирует полный pipeline end-to-end.
