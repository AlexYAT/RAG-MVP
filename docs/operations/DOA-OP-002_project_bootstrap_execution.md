# Title

Project bootstrap execution

## 1. Context

- Документ — первый исполняемый шаг в рамках **DOA-OP-001** (MVP initial execution plan).
- Покрывает **Phase 1 — Project bootstrap**.
- Цель этапа: создать минимальную рабочую основу проекта, достаточную для последующего подключения данных и pipeline без блокирующих технических долгов на уровне «проект не стартует».

## 2. Objective

- Система (backend) должна **запускаться** в локальной среде.
- Должен существовать **backend skeleton**: единая точка входа приложения на стороне сервера.
- Должна быть **точка входа API** — минимальный контракт «клиент → сервер», подтверждающий работоспособность сервиса.

## 3. Scope

**Включить:**

- Базовая структура каталогов и границ модулей проекта.
- Backend-приложение с явным entrypoint.
- Минимальный API: один служебный endpoint (например health / test — по смыслу проверки «живости»).
- Возможность запуска через локальный сервер разработки.

**Не включать:**

- Retrieval и индексацию.
- Ingest и работу с датасетом.
- Генерацию ответов и подключение моделей.
- Сценарную логику (FAQ / selection / overview).
- Web UI и любой пользовательский интерфейс.

## 4. Tasks

На уровне намерений, без кода и без привязки к стеку:

1. **Структура проекта** — определить и зафиксировать базовую раскладку репозитория под backend и будущее расширение (без реализации RAG).
2. **Основной entrypoint backend** — одна явная точка запуска серверного приложения.
3. **Минимальный API endpoint** — один маршрут, по которому можно убедиться, что приложение принимает HTTP-запросы и отвечает предсказуемо.
4. **Запуск приложения** — воспроизводимая команда или сценарий запуска локального сервера для разработки.

## 5. Expected Result

- Приложение **запускается локально** без обходных действий вне описанного bootstrap.
- **Доступен** описанный минимальный endpoint.
- По ответу endpoint можно **проверить**, что система «живая» (сервис поднят и обрабатывает запросы).

## 6. Validation

Критерии приёмки этого шага:

- Сервер **стартует без ошибок**, препятствующих дальнейшей разработке.
- Endpoint **отвечает** успешно в ожидаемом смысле проверки доступности.
- Запуск и проверка **не требуют** наличия dataset, retrieval, генерации или UI.

## 7. Out of Scope

- Любые бизнес-функции и предметная логика домена.
- RAG pipeline целиком или частично.
- LLM, эмбеддинги, векторное хранилище — как обязательные зависимости этапа.
- UI, статика, фронтенд-сборка.
